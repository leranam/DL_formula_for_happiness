# Digital Happiness Index: Анализ ментального благополучия пользователей через Deep Learning и цифровой след

## Описание проекта
Данный проект посвящен разработке **Индекса цифрового счастья** — комплексной метрики ментального состояния пользователя социальной сети ВКонтакте. В основе проекта лежит семантический анализ текстов постов (NLP), исследование структуры социальных связей (Graph Analysis) и выявление негативных паттернов поведения.

Проект реализован в рамках курса по **Deep Learning** и включает в себя сравнение классических методов машинного обучения, современных трансформеров (RuBERT) и новейших рекуррентных архитектур (xLSTM).

---

##  Технологический стек
*   **Язык:** Python 3.10+
*   **DL-фреймворки:** PyTorch, HuggingFace Transformers
*   **Оптимизация:** Optuna (Bayesian Optimization)
*   **Классический ML:** Scikit-learn (LinearSVC, Logistic Regression)
*   **Анализ данных:** Pandas, NumPy, NetworkX (графовые метрики)
*   **Визуализация:** Seaborn, Matplotlib

---

##  Пайплайн проекта

Проект разбит на 6 этапов (ноутбуков), формирующих единую систему обработки данных:

1.  **01_task_data_eda_preprocessing**: Сбор данных, фильтрация профилей (18-35 лет), генерация графовых признаков (коэффициент кластеризации, связность) и первичная очистка текстов.
2.  **02_baseline_tfidf_logreg**: Создание базовой линии качества (Baseline) с использованием TF-IDF и логистической регрессии. Фиксация единого Train/Test разбиения.
3.  **03_tfidf_linearsvc_cv**: Обучение LinearSVC и внедрение **Cross-Validation Pipeline**, исключающего утечку данных (Data Leakage) при векторизации.
4.  **04_finetune_rubert_pytorch**: Полноценный **Fine-tuning RuBERT** (`DeepPavlov/rubert-base-cased`) на PyTorch. Реализован кастомный цикл обучения, использование AMP (смешанная точность) и сохранение лучших чекпоинтов.
5.  **05_rubert_optuna_xlstm**: 
    *   Автоматический подбор гиперпараметров (Learning Rate, Dropout) через **Optuna**.
    *   Реализация и обучение экспериментальной архитектуры **xLSTM (sLSTM)** с экспоненциальным гейтированием и нормализацией состояний.
6.  **06_model_comparison_vk_happiness**: Финальное сравнение моделей, кросс-доменное тестирование на датасете **RuSentiTweet**, ручная валидация на данных VK и расчет итогового Индекса Счастья.

##  Математическая модель индекса
Итоговый индекс счастья рассчитывается по формуле:

$$HappinessIndex = clip(BaseIndex \cdot P_{toxic} \cdot P_{showcase} - D_{insomnia}, 0, 1)$$

Где:
*   **BaseIndex:** 50% Sentiment (DL-модель) + 30% Social Network (графы) + 20% Activity.
*   **P_toxic:** Понижающий коэффициент (0.85) за участие в депрессивных сообществах.
*   **P_showcase:** Штраф (0.90) за «эффект витрины» (неискренность профиля).
*   **D_insomnia:** Вычет (до -0.15) за ночную активность (маркер стресса/бессонницы).

##  Ключевые особенности (DL Highlights)
*   **xLSTM (2024):** Внедрение новейшей рекуррентной архитектуры, конкурирующей с трансформерами.
*   **Hyperparameter Tuning:** Использование байесовского поиска параметров для достижения максимальной обобщающей способности RuBERT.
*   **Model Checkpointing:** Система защиты от переобучения (overfitting), автоматически выбирающая лучшую версию модели.
*   **Cross-Domain Validation:** Проверка устойчивости моделей на данных из другой социальной сети (Twitter/RuSentiTweet).
